# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Erythos, please report it **privately** rather than opening a public issue.

Preferred channels (in order):

1. GitHub's [private vulnerability reporting](https://github.com/wolfuardian/erythos/security/advisories/new)
2. Email: wolfuardian@gmail.com

Please include:

- A description of the issue
- Steps to reproduce (or a minimal proof of concept)
- Affected versions, if known
- Suggested mitigation, if any

We aim to acknowledge reports within 7 days.

## Supported Versions

Erythos is in active development. Security fixes are applied to the latest version on `master`. There are no long-term support branches at this time.

## Disclosure

After a fix is released, we will publish a brief advisory and credit the reporter (with consent).
