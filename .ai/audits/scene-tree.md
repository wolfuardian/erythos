# Scene Tree 視覺審計報告

審計日期：2026-04-17
截圖：overview（5 個節點未選取）、hover（Cube row）、selected（Cube row）
整體印象：功能已在，但整體感像還沒收尾的 v1 prototype——hover 幾乎無感、選取強度不夠果斷、badge 色彩跨類別撞色嚴重、section header 細到像水印。這個 panel 每天高頻使用，現在的視覺回饋力道不足以支撐工作流。

---

## 視覺問題

### 字級 / 排版

- [ ] "SCENE" section header 字極小、顏色接近背景，在視覺上幾乎消失——使用者掃視時難以確認自己在哪個區塊，分量完全不像一個區塊標題，更像浮水印。
- [ ] 物件名稱文字（Cube / Sphere / Camera 等）在 overview 狀態下所有 row 的字重、字色完全一致，沒有任何視覺層級差異——幾何、燈光、相機語意不同，文字卻完全等量，資訊密度雖高但辨識速度慢。

### 空間節奏

- [ ] Row 高度（約 22px）配上左側 badge + 文字，垂直 padding 非常緊，呼吸感幾乎為零。5 個 row 疊在一起像一塊實心磚，掃讀時眼睛容易在相鄰 row 之間滑動。
- [ ] Panel 下半段是大片空白，與上方緊密的 row 群形成強烈落差——整體重心過度集中在頂部，空間分佈不均衡。

### 色彩 / 調性

- [ ] Cube（B badge，amber 橙黃）、Sphere（S badge，橙紅）、Group（G badge，橙）三個 badge 色系高度重疊，同一視野內三個橙色系 badge 並排，類別辨識失效——使用者無法一眼靠 badge 色判斷物件類型。
- [ ] Hover 狀態（`--bg-hover` #383838 對 `--bg-panel` #242424）亮度差約 15 個灰階步進，在深色背景下極不顯眼。滑鼠移到 row 上幾乎感覺不到任何反饋，互動感知薄弱。
- [ ] Hover 與 Selected 的色差在整體暗色調中辨識困難——`--bg-hover`（偏暖灰）vs `--bg-selected`（#2a4060 冷藍）理論上有色溫差，但明度差距小，快速操作時容易混淆兩個狀態。

### 狀態層級

- [ ] Selected 狀態（#2a4060 藍底）視覺權重偏弱——在深色 UI 中，這個藍色底板亮度提升幅度有限，不夠果斷。選中的 row 應該是整個 panel 視覺焦點，現在感覺只是「略微不同的 row」，而非「被選中的物件」。
- [ ] Hover 反饋強度太弱（接近無感），缺乏即時互動感——使用者在 hover 瞬間沒有足夠的視覺確認，不確定游標是否正確對準 row。

### 細節品質

- [ ] Badge 是帶圓角的實心色塊，邊角 radius 目測為 `--radius-sm`（2px）或略大，但 Cube（B）的 badge 呈現出偏方整形、Sphere（S）的 badge 比例略有不同——各 badge 外觀尺寸微妙不一，視覺上不整齊。
- [ ] Badge 內的字母（B / S / C / G / L）字級小，部分字母在深色 badge 上的對比讀取稍費力（特別是 L 在黃底上）。

### 整體感

- [ ] 單看這個 panel 像是功能完成但視覺未收尾的工具原型——沒有明確的設計語言在 row 狀態間流動，hover / selected / idle 三態差異太小，讓整個 panel 感覺是「貼上去的清單」而非「精心設計的場景導航器」。
- [ ] 與 3D 編輯器「每日高頻使用工具」的定位不符——高頻使用的 panel 需要讓使用者能快速且自信地辨識狀態，目前的視覺精度不足以支撐這個要求。
