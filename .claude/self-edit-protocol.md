# AH 自改 CLAUDE.md 協定

CLAUDE.md 是 AH 主指令來源；自改 = 改自己的指令。走 meta-exception（直 commit main，不走 PR），但需自律。

## 前提檢查（任一不滿足即停）

- 改動非「為當前任務開後門」（避免 in-flight 鬆綁）
- 涉及鬆綁時，先列「現行規則為何不適用」
- 確認走 meta-exception（直改 main）

## 修改類型與風險

| 類型 | 判準 | 處理 |
|------|------|------|
| T1 事實更新 | 路徑、版本、指令變更 | 直改 |
| T2 去冗餘 | 刪失效規則、合併重複段 | 直改 |
| T3 新增規則 | 加新行為限制或流程 | 直改 |
| T4 鬆綁規則 | 移除 / 放寬既有限制 | **先提案後改**：列移除 X / 為何安全 / 是否有替代保護 → 等指揮家確認 |

## 動作約束

- 單次 session 最多 5 個獨立項目，超過分次
- 每項要能用一句話解釋動機（commit message 說「為何改」非「改了什麼」）
- T2 / T4 在 `.claude/knowledge.md` 留一筆：`YYYY-MM-DD：<動作> X，因 Y`

## 自我審查（改完逐條過）

- 與既有段落 authority 是否衝突？衝突要明確指定勝者
- 細節該放文件地圖（外部 .md）還是內聯？預設外部，CLAUDE.md 只放 pointer
- 新規則用正面目的語氣（4.7 指令原則）
- 刪除時：檢查其他 skill / 模組 CLAUDE.md 是否仍引用該規則
- 改動後 CLAUDE.md 總長度沒暴漲（地圖非百科）

## 中止條件（任一觸發即停回報指揮家）

- 規則散落 3+ 處（先 refactor 再改內容）
- 改動會影響 skill 行為（先改 skill 不改 CLAUDE.md）
- 改動動機是「當前任務方便」而非「長期正確」
